# General Branch

Use this branch when the task is not clearly coding, writing, or research.

## Branch Rules

- Keep the task framed around the user's goal, current constraints, and a conservative success definition.
- Prefer short todo tracking for simple work and minimal internal plans for anything that may drift across turns.
- Use soft pause only for consequential but non-hard-stop questions, and only when a brief answer would materially reduce rework.
- Before claiming completion, verify that the output is usable, bounded, and clearly handed off.
